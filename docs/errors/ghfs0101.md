# GHFS0101: Operation conflict — remote was updated

Code: `GHFS0101`
Level: error

## What this error means

`ghfs` detected that the issue or pull request you are about to modify has been updated on GitHub since the last local sync. Applying your queued operation could clobber a change you haven't seen yet, so the execute step stops.

## Why this happens

- Someone else commented, labelled, or changed state on the issue/PR since your last `ghfs sync`
- A bot (e.g. CI, Dependabot) updated the PR
- Your local tracked copy under `.ghfs/` is older than the remote

## How to fix it

Refresh local state, then re-run the operation:

```bash
ghfs sync
ghfs execute
```

After syncing, review the updated item in `.ghfs/issues/**` or `.ghfs/pulls/**` to confirm your operation still makes sense. If the remote change already did what you intended, drop the op from the queue.

## Additional context

- `remote updated_at` in the message is the timestamp from the GitHub API at the moment the conflict was detected
