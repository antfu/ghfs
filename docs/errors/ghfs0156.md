# GHFS0156: Action requires a non-empty comment

Code: `GHFS0156`
Level: warn

## What this warning means

A `comment` or `close-with-comment` action was written without any body text after the issue reference, or with an empty quoted string. The line is skipped.

## Why this happens

- Forgot to add the comment body
- Provided only empty quotes: `comment #126 ""`
- Stripped the body while editing

## How to fix it

Provide non-empty comment text, quoted if it contains spaces:

```md
# Wrong
comment #126
comment #126 ""

# Correct
comment #126 "Thanks for reporting!"
close-comment #126 "Closing as completed"
```

## Additional context

- Comment bodies support Markdown — GitHub will render it the same way as comments entered via the web UI
