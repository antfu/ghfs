# GHFS0011: Invalid repo in `ghfs.config.ts`

Code: `GHFS0011`
Level: error

## What this error means

The `repo` value in `ghfs.config.ts` could not be parsed as a GitHub repository reference. `ghfs` accepts `owner/name`, a full GitHub URL, or a git remote URL.

## Why this happens

- The `repo` field is set to just `"ghfs"` instead of `"antfu/ghfs"`
- A typo or extra slash snuck into the value
- The value points to a non-GitHub URL

## How to fix it

```ts
// ghfs.config.ts
import { defineConfig } from '@ghfs/cli'

export default defineConfig({
  repo: 'antfu/ghfs', // owner/name
  // or: 'https://github.com/antfu/ghfs'
  // or: 'git@github.com:antfu/ghfs.git'
})
```
