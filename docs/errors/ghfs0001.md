# GHFS0001: Missing GitHub token

Code: `GHFS0001`
Level: error

## What this error means

`ghfs` could not find a GitHub token to authenticate API requests. Every call to the GitHub REST and GraphQL APIs requires a token, so the command cannot proceed without one.

## Why this happens

- Neither `GH_TOKEN` nor `GITHUB_TOKEN` is set in the environment
- The GitHub CLI (`gh`) is not logged in, or `gh auth token` returned nothing
- The command was run in an environment (CI, container, sandbox) that did not inherit your local shell variables
- The prompt to enter a token was skipped with `--non-interactive`

## How to fix it

Provide a token via one of these methods:

```bash
# 1) Export an environment variable
export GH_TOKEN=ghp_xxx

# 2) Or log in with the GitHub CLI
gh auth login

# 3) Or let ghfs prompt you interactively
ghfs
```

Any [personal access token](https://github.com/settings/tokens) or fine-grained token with `repo` scope works. For CI, store the token as a secret and expose it as `GH_TOKEN`.
