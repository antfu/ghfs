# GHFS0100: Interactive execute prompts are unavailable

Code: `GHFS0100`
Level: error

## What this error means

`ghfs execute` needs to confirm or prompt for destructive operations, but the current run is non-interactive (e.g. piped, CI, `--non-interactive`) so no prompt can be shown.

## Why this happens

- `ghfs execute --non-interactive` was invoked without already-approved ops
- The command is running in CI or under a tool that doesn't allocate a TTY
- stdin/stdout is redirected

## How to fix it

Run the command interactively:

```bash
ghfs execute
```

Or pre-approve and execute the queue programmatically. Review `execute.md` / `execute.yml` ahead of time so no extra prompts are needed, then re-run with `--non-interactive` once the content is final.
