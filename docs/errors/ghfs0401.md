# GHFS0401: Missing tracked canonical data for issue/PR

Code: `GHFS0401`
Level: error

## What this error means

`ghfs` tried to reconcile a per-item edit with its tracked canonical copy (the last known GitHub state), but no canonical data was found for that issue. Without the baseline, `ghfs` can't tell what's actually changing, so the sync step stops.

## Why this happens

- The item was never successfully synced to `.ghfs/` (network error mid-sync, process killed)
- The tracked state files were hand-edited or deleted
- Two `ghfs` clients wrote to the same `.ghfs/` in parallel and one clobbered the other
- Upgrading `ghfs` changed the tracked-state format and the cache hasn't been refreshed

## How to fix it

Re-sync the item from GitHub:

```bash
ghfs sync
```

If the problem persists, delete tracked state for that specific item (or the whole `.ghfs/` tree) and re-run `ghfs sync`. Your per-item markdown edits stay intact — only the canonical snapshot is rebuilt.
