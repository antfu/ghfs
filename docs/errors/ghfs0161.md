# GHFS0161: Per-item edit has invalid or missing frontmatter

Code: `GHFS0161`
Level: warn

## What this warning means

A per-item markdown file (`.ghfs/issues/**` or `.ghfs/pulls/**`) was found, but its YAML frontmatter could not be parsed or is missing entirely. `ghfs` can't diff the file against tracked state, so per-item operations for that issue are skipped.

## Why this happens

- The frontmatter block was accidentally edited or deleted
- A trailing `---` was removed, leaving the YAML unterminated
- A YAML syntax error inside the frontmatter (unbalanced quotes, bad indentation)
- An editor stripped the frontmatter as "unused metadata"

## How to fix it

Make sure the file begins with a valid YAML frontmatter block:

```md
---
title: Some bug
state: open
labels: [bug]
assignees: [antfu]
milestone: v1
---

Issue body goes here.
```

Supported fields: `title`, `state` (`open` / `closed`), `labels`, `assignees`, `milestone`. If you want to abandon the local edits, re-sync:

```bash
ghfs sync
```
