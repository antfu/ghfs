# GHFS0150: Invalid quoted string syntax

Code: `GHFS0150`
Level: warn

## What this warning means

While parsing `execute.md`, `ghfs` found a line containing a quoted string it could not parse — for example, an unterminated quote or an unsupported escape. The line is ignored.

## Why this happens

- Missing closing quote: `set-title #125 "New title`
- Mixed quote styles: `set-title #125 "New title'`
- Unescaped inner quotes: `comment #1 "He said "hi""`

## How to fix it

Use matching `"..."` or `'...'` around the text. Escape inner quotes or switch quote style:

```md
# Wrong
set-title #125 "He said "hi""

# Correct
set-title #125 'He said "hi"'

# Also correct
set-title #125 "He said \"hi\""
```

## Additional context

- This is a warning: parsing continues and other operations in the file still apply
