# GHFS0010: Invalid `--repo` value

Code: `GHFS0010`
Level: error

## What this error means

The value passed to the `--repo` CLI flag could not be parsed as a GitHub repository reference. `ghfs` accepts three forms: `owner/name`, a full GitHub URL, or a git remote URL.

## Why this happens

- Typo in the slug (e.g. `antfu/` with no name, or extra slashes)
- Pasted a URL from a non-GitHub host
- Passed just the repo name without the owner (`ghfs` instead of `antfu/ghfs`)
- Copy/paste included quotes or trailing whitespace that made the value unparseable

## How to fix it

Use one of the supported forms:

```bash
# owner/name slug (preferred)
ghfs --repo antfu/ghfs

# full GitHub URL
ghfs --repo https://github.com/antfu/ghfs

# git remote URL
ghfs --repo git@github.com:antfu/ghfs.git
```
