# GHFS0203: Cannot remove a per-item edit from the queue

Code: `GHFS0203`
Level: error

## What this error means

The queue panel tried to remove an operation that comes from a per-item markdown edit (a frontmatter change in `.ghfs/issues/**` or `.ghfs/pulls/**`). Per-item ops aren't stored in a queue file — they're generated by diffing the markdown file against tracked state — so they can't be "removed" from the queue panel.

## Why this happens

- You clicked "remove" on a queue row whose `source` is the per-item markdown file
- You sent a delete request to the API for a per-item op id

## How to fix it

Edit the target file directly to undo or change the per-item edit:

1. Open the file named in the error (e.g. `.ghfs/issues/00125-some-bug.md`)
2. Revert the frontmatter field you want to drop from the queue
3. Save — the op will automatically disappear from the queue
