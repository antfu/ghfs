# GHFS0002: Token prompt cancelled

Code: `GHFS0002`
Level: error

## What this error means

`ghfs` tried to prompt you for a GitHub token because none was found in the environment, but the prompt was dismissed (Ctrl-C, ESC, empty submission). Without a token, the command cannot authenticate to GitHub.

## Why this happens

- You hit Ctrl-C / ESC at the token prompt
- The prompt was rendered but stdin was closed before a value was entered
- The command is running non-interactively and can't actually read from a TTY

## How to fix it

Set a token ahead of time so the prompt is skipped entirely:

```bash
export GH_TOKEN=ghp_xxx
ghfs
```

Or use the GitHub CLI:

```bash
gh auth login
ghfs
```

If you're running in CI or a script, always set `GH_TOKEN` / `GITHUB_TOKEN` — interactive prompts will not work there.
