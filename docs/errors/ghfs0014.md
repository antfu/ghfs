# GHFS0014: Repo mismatch detected

Code: `GHFS0014`
Level: error

## What this error means

The `origin` git remote and the `repository` field in `package.json` point to different GitHub repositories. `ghfs` cannot tell which one you want to operate on.

## Why this happens

- The project was forked but `package.json` still references the upstream repo
- `package.json` was copy-pasted from another project and the `repository` field wasn't updated
- The git remote was changed (e.g. renamed owner) but `package.json` wasn't updated

## How to fix it

Pick one source of truth, or disambiguate with `--repo`:

```bash
# disambiguate on the command line
ghfs --repo antfu/ghfs
```

Or set `repo` explicitly in `ghfs.config.ts`:

```ts
export default defineConfig({
  repo: 'antfu/ghfs',
})
```

Or fix the underlying mismatch — update `package.json`'s `repository` field or change the git remote so both agree.
