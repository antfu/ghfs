# GHFS0160: Per-item edit missing markdown file

Code: `GHFS0160`
Level: warn

## What this warning means

`ghfs execute` expected a per-item markdown file for an issue or PR (under `.ghfs/issues/**` or `.ghfs/pulls/**`) but the file at the recorded path no longer exists. Per-item operations for that issue are skipped.

## Why this happens

- The file was deleted or renamed after `ghfs sync` wrote it
- The issue was moved between `open/` and `closed/` without a follow-up sync
- A filesystem case-sensitivity mismatch (macOS vs Linux) hid the file

## How to fix it

Re-run sync so local state matches what's on disk:

```bash
ghfs sync
```

If the file was intentionally removed, there's nothing to do — the warning just tells you that per-item ops for that issue won't be generated.
