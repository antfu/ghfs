# GHFS0153: Action expects a single issue reference

Code: `GHFS0153`
Level: warn

## What this warning means

The action can only target one issue/PR at a time (e.g. `set-title`, `comment`, `close-with-comment`), but the line either has no `#n` reference or has more than one. The line is skipped.

## Why this happens

- Multiple `#n` tokens on a line that should name exactly one
- No `#n` reference at all
- Missing the `#` prefix in front of the number

## How to fix it

Use one `#n` reference:

```md
# Wrong
set-title #125 #126 "New title"

# Correct — one per line
set-title #125 "New title"
set-title #126 "Another title"
```

## Additional context

- For actions that do accept multiple targets, see [GHFS0157](./ghfs0157.md)
