# GHFS0300: Unexpected patch response for pull request

Code: `GHFS0300`
Level: error

## What this error means

`ghfs` requested a pull request's patch (the unified diff that backs `.ghfs/pulls/*.patch`) from GitHub and received a response in an unexpected shape. The patch could not be stored.

## Why this happens

- GitHub returned an HTML page instead of the patch (rate-limited, auth challenge, transient outage)
- The PR is too large and GitHub refused to serve the raw patch
- A proxy or VPN rewrote the response
- GitHub changed the raw patch endpoint in a way `ghfs` didn't handle

## How to fix it

- Try `ghfs sync` again — this is often transient
- Verify your token still works: `gh auth status` or `curl -H "Authorization: Bearer $GH_TOKEN" https://api.github.com/user`
- If the PR is huge, open it in the browser to see what GitHub actually returns
- If it keeps happening for a specific PR, file an issue with the PR number on [antfu/ghfs](https://github.com/antfu/ghfs/issues)

## Additional context

- Patches live at `.ghfs/pulls/<number>-*.patch`
