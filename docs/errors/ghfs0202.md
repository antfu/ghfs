# GHFS0202: Queue entry not found

Code: `GHFS0202`
Level: error

## What this error means

The UI or API referenced a queue entry by id, but no entry with that id exists in the current queue.

## Why this happens

- The queue was rewritten (e.g. `execute.md` was edited) and old ids are no longer valid
- A previous execute run already consumed that entry
- A stale UI page is holding on to an id that no longer exists
- Two clients raced and one removed the entry first

## How to fix it

Refresh the UI (or re-fetch the queue) to get the current list of entries, then retry. If you invoked the API directly, list the queue again and pick a fresh id.

## Additional context

- Queue entries are derived from `execute.md`, `execute.yml`, and per-item edits — any of those changing invalidates in-flight ids
