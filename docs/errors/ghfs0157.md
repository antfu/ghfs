# GHFS0157: Action expects one or more issue references

Code: `GHFS0157`
Level: warn

## What this warning means

An action that can target many issues/PRs at once (e.g. `close`, `reopen`) was written with no `#n` references. The line is skipped.

## Why this happens

- Forgot to list any `#n` after the action
- References don't start with `#`
- The only token on the line was the action keyword

## How to fix it

Add one or more `#n` references, space-separated:

```md
# Wrong
close

# Correct
close #123
close #123 #234 #235
```

## Additional context

- For actions that only accept a single target, see [GHFS0153](./ghfs0153.md)
