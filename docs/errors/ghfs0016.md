# GHFS0016: Invalid repo slug

Code: `GHFS0016`
Level: error

## What this error means

An internal function received a repository slug that is not in `owner/name` form. This typically means the slug was constructed or stored incorrectly somewhere along the way.

## Why this happens

- A URL or remote string was passed directly to a function that expects an already-parsed `owner/name` slug
- Tracked state (under `.ghfs/`) is malformed
- A custom caller of the `ghfs` APIs passed an unparsed value

## How to fix it

Ensure the slug is exactly `owner/name`:

```ts
// Correct
resolveRepo('antfu/ghfs')

// Wrong — pass through the slug parser first
resolveRepo('https://github.com/antfu/ghfs')
```

If you hit this from the CLI, try re-running with an explicit `--repo`:

```bash
ghfs --repo antfu/ghfs
```
